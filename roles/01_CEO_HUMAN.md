# Papel: CEO (Chief Executive Officer) - Humano

## Responsabilidade Principal

**Manter a visão de negócio, priorizar valor e orquestrar a colaboração entre especialistas técnicos (CTO e Dev) para garantir que o sistema construído atenda objetivos reais.**

---

## Atribuições

| Categoria | Atribuições Específicas | Quando | Output |
|-----------|------------------------|---------|--------|
| **Visão Estratégica** | Definir objetivos de negócio do projeto | Início do projeto | Documento de visão |
| | Identificar usuários-alvo e suas necessidades | Planejamento | User personas |
| | Estabelecer critérios de sucesso mensuráveis | Planejamento | KPIs/métricas |
| **Priorização** | Criar e manter backlog priorizado | Contínuo | Backlog ordenado |
| | Decidir o que entra em cada sprint | Início de sprint | Sprint plan |
| | Aprovar ou rejeitar mudanças de escopo | Quando solicitado | Decisão go/no-go |
| **Orquestração** | Facilitar comunicação entre CTO e Dev | Conforme necessário | Reuniões/mensagens |
| | Remover impedimentos não-técnicos | Quando identificados | Ações de desbloqueio |
| | Garantir alinhamento com objetivos | Contínuo | Check-ins |
| **Validação** | Revisar entregas contra critérios de aceitação | Final de sprint | Checklist validado |
| | Testar funcionalidades em cenários reais | Após implementação | Bugs/feedback |
| | Aprovar ou solicitar ajustes | Após revisão | Aprovação formal |
| **Contexto de Negócio** | Fornecer informações sobre mercado/usuários | Quando relevante | Briefings |
| | Esclarecer requisitos ambíguos | Quando questionado | Clarificações |
| | Validar premissas técnicas com realidade do negócio | Durante especificação | Feedback |

---

## Limitações (O Que o CEO NÃO Faz)

### ❌ Não Especifica Implementação

O CEO define **O QUÊ** e **POR QUÊ**, não **COMO**.

### ❌ Não Escreve Código
### ❌ Não Toma Decisões Arquiteturais
### ❌ Não Microgerencia
